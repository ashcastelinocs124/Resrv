# Resrv

Custom queue management system for the SCD (Student Creative Design) facility at the University of Illinois. Replaces Waitwhile with an autonomous, Discord-native queue built for maker spaces.

## Why

The existing system (Waitwhile) is expensive per-queue, manually operated, and progressively paywalls analytics and branding. Staff are forced to juggle queue management, user training, and safety monitoring simultaneously. Time-based reservations were tried and abandoned — users either overrun or leave machines idle.

Resrv eliminates the manual queue work entirely with an autonomous FIFO agent, while giving staff full override control when they need it.

## Features

- **Discord Bot** — users join queues via button interactions in Discord. No app downloads, no accounts to create beyond Discord + Illinois email verification.
- **Autonomous FIFO Agent** — background loop auto-transitions users through waiting → serving → completed. Sends 30-minute reminders. Handles no-shows with a 10-minute grace period.
- **Staff Web Dashboard** — real-time React SPA showing all machine queues with drag-and-drop reordering, manual overrides, and action buttons.
- **AI Analytics** — daily and weekly usage summaries generated by OpenAI, posted to a staff Discord channel. Visual charts (jobs/day, wait times, peak hours, completion rates) on the web panel.
- **Email Verification** — Illinois email (@illinois.edu) required to join. Toggleable for public events.
- **Manual Override** — staff can bump, reorder, remove, or skip users at any time via Discord commands or the web panel.

```
│  └──────┬───────┘       └────────┬───────────┘ │

└─────────────────────────────────────────────────┘
```

## Queue Flow

```
User clicks [Join Queue] on Discord embed
       │
       ▼
  ┌─ Verified? ─┐
  │ No           │ Yes
  ▼              ▼
DM: verify    Added to queue
email → code  (bot confirms position)
  │
  ▼
Verified → added to queue
       │
       ▼
Agent auto-serves next user when machine is free
       │
       ▼
Bot DMs: "You're up! Head to [machine]"
       │
       ▼
30-min reminder: "Still using it?"
       │
       ├─ ✅ Confirm → stays in serving
       ├─ ❌ Done → complete + feedback
       └─ No response (10 min) → auto-complete as no_show
```

## Staff Commands (Discord)

| Command | Description |
|---------|-------------|
| `/bump @user` | Move user to top of queue |
| `/remove @user` | Remove from queue |
| `/skip @user` | Mark as no-show, advance |
| `/pause <machine>` | Pause a machine queue |
| `/public on\|off` | Toggle public mode (skip email verification) |

## Setup

### Prerequisites

- Python 3.11+
- Node.js 18+
- A Discord bot token ([Discord Developer Portal](https://discord.com/developers/applications))
- OpenAI API key
- Resend API key (free tier) for email verification

### Installation

```bash
# Clone the repo
git clone https://github.com/Agentic-AI-UIUC/Resrv.git
cd Resrv

# Backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Frontend
cd web
npm install
npm run build
cd ..

# Configure environment
cp .env.example .env
# Edit .env with your Discord token, OpenAI key, Resend key, etc.

# Run
python main.py
```

### Environment Variables

| Variable | Description |
|----------|-------------|
| `DISCORD_TOKEN` | Discord bot token |
| `OPENAI_API_KEY` | OpenAI API key for analytics |
| `RESEND_API_KEY` | Resend API key for email verification |
| `ADMIN_PASSWORD` | Staff dashboard login password |
| `DATABASE_PATH` | Path to SQLite database (default: `./data/resrv.db`) |
